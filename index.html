生成树协议的基本原理

基本思想：在网桥之间传递特殊的消息（配置消息），包含足够的信息做以下工作：

–> 从网络中的所有网桥中，选出一个作为根网桥（Root）

–> 计算本网桥到根网桥的最短路径

–> 对每个LAN，选出离根网桥的最近的那个网桥作为指定网桥，负责所在LAN的数据转发

–> 网桥选择一个根端口，该端口给出的路径是此网桥到根网桥的最佳路径

–> 选择除根端口之外的包含于生成树上的端口（指定端口）

端口角色 描述
Root Port 根端口，是所在交换机上离根交换机最近的端口，处于转发状态。
Designated Port 指定端口，转发所连接的网段发往根交换机方向的数据和从交换机方向发往所连接的网段的数据。
Alternate Port 预备端口，不向所连网段转发任何数据。

Windows：

*.txt文件：普通的文本文档（普通文件）

由此可知，在Windows操作系统中，我们可以通过文件的扩展名来区分文件的类型。但是，在Linux系统中，其不是通过扩展名来区分文件类型的！

Linux 系统通过==权限位标识==来确定文件类型，常见的文件类型有普通文件、目录、链接文件、块设备文件、字符设备文件等几种。Linux 的可执行文件就是普通文件被赋予了可执行权限。(文件权限重点讲解)

这样的文件扩展名主要有以下几种：
压缩包：Linux 下常见的压缩文件名有 .gz、.bz2、*.zip、*.tar.gz、*.tar.bz2、*.tgz 等。

程序文件：Shell 脚本一般用 “.sh” 扩展名结尾。
网页文件：网页文件一般使用 “.php” 等结尾，不过这是网页服务器的要求，而不是 Linux 的要求。

5、Linux系统的文件目录结构

☆ 树状结构

Linux 系统不同于 Windows，没有 C 盘、D 盘、E 盘那么多的盘符，只有一个==根目录（/）==，所有的文件（资源）都存储在以==根目录（/）==为树根的==树形==目录结构中。

在 Linux 根目录（/）下包含很多的子目录，称为一级目录。
例如 bin、boot、dev 等。
同时，各一级目录下还含有很多子目录，称为二级目录。
例如 /bin/bash、/bin/ed 等。

问题：为什么要了解Linux系统文件目录的用途呢？

一级目录(/根目录下的子目录) 作用（用途）
/bin与/sbin目录 系统命令文件目录，/bin普通命令，/sbin超级命令（只能root账号使用）

/media目录 多媒体目录，用于挂载软盘、光盘
/mnt目录 挂载目录。早期 Linux 中只有这一个挂载目录，并没有细分。系统建议这个目录用来挂载额外的设备，如 U 盘、移动硬盘和其他操作系统的分区
/opt目录与/usr目录 /opt目录第三方软件保存目录，手工安装的源码包软件都可以安装到这个目录中。但是大多数运维工程师喜欢把软件放置于/usr目录，相当于Windows中的Program Files。
/run目录 运行时目录，CentOS7中新增目录，把运行时的程序放置于此目录
/tmp目录 临时文件目录
/var目录 经常变化的目录，网页文件、数据库文件、日志文件都放置于此

问题：用超级管理员设置文档的权限命令是# chmod -R 731 shop，请问这个命令有没有什么不合理的地方？

731 = 7 3 1

问题在权限731中的3权限，3表示写+执行权限，但是写又必须需要能打开之后才可以写，因此必须需要具备可读权限，因此此权限设置不合理。

1、NTP服务介绍

NTP是网络时间协议(Network Time Protocol)，它是用来同步网络中各个计算机的时间的协议。

工作场景：

在NTP中，定义了时间按照服务器的等级传播，Stratum层的总数限制在15以内

二、YUM包管理工具

1、Linux软件的安装方式

Linux软件的安装方式：① RPM安装 ② YUM安装 ③ 源码编译安装

RPM：必须先下载*.rpm的软件包

优点：安装简单，直接通过rpm -ivh就可以实现对软件的安装操作

缺点：在安装或卸载时可能需要解决软件的依赖关系

YUM：不需要提前下载软件，只要有网络的支持，其会自动下载并安装软件

优点：安装简单，操作方便，更重要的是其不需要解决软件的依赖关系

缺点：一般情况下，其获取的软件版本可能并不是最新版本

2、YUM概述

是一个在Fedora和RedHat以及CentOS中的Shell前端软件包管理器。

YUM本身还是基于rpm包管理，能够从指定的服务器(yum源）自动下载RPM包并且安装，可以==自动处理依赖性关系==，并且==一次安装所有依赖的软件包==，无须繁琐地一次次下载、安装。

3、YUM的运行原理

由上图可知，当我们执行yum install 软件名称系统会自动到对应的YUM仓库下载对应的软件，如果软件存在依赖关系，其也会自动从仓库下载并安装对应的依赖软件。

但是由于CentOS默认使用到YUM仓库主要来源于CentOS官方网站，所以其下载速度可能会非常慢。

中国网络下载 => 网络 => 长城防火墙（屏蔽了国外的一些资源，Google、Youtube、Facebook…） => 国外的网站资源
